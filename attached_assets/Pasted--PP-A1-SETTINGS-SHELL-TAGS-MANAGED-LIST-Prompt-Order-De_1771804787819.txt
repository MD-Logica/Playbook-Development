## 2. TAGS MANAGED LIST тАФ SUB-PAGE

### Purpose
Tags are labels applied to patient/contact records for segmentation and reporting. Staff select from predefined tags rather than typing freeform. All tags are reportable.

### Page Header
- Title: "Tags"
- Subtitle: "Create and manage tags applied to patient records. Tags are used for segmentation and reporting."
- Primary CTA button: "+ Add Tag" (top right)

### Tag List Display
Each tag displays as a table row with:
- **Color swatch** тАФ small filled circle showing assigned color
- **Tag name**
- **Status badge** тАФ "Active" or "Archived" (only visible when Show Archived is toggled on)
- **In Use count** тАФ number of patient records this tag is applied to (e.g., "14 patients")
- **Actions** тАФ Edit (pencil icon), Archive (archive icon). No direct delete on active items.

Default view shows only Active tags.

### "Show Archived" Toggle
- Positioned above the table, right-aligned
- When on: archived tags appear with muted/greyed styling and an "Archived" badge
- Archived tags show "Restore" action instead of "Archive"
- Archived items also show "Delete Permanently" as a secondary destructive action (only on archived items, never on active ones)

### Add / Edit Tag тАФ Modal
Fields:
- **Tag Name** тАФ text input, required, max 32 characters
- **Color** тАФ curated palette of 12 color circles, selectable. Selected color shows a checkmark or ring indicator. Palette: Slate, Red, Orange, Amber, Green, Teal, Cyan, Blue, Indigo, Violet, Pink, Rose
- **Save** / **Cancel**

Validation: Tag name must be unique (case-insensitive). Show inline error if duplicate.

### Archive Behavior
- If tag has 1+ patients: show confirmation dialog тАФ "Archive [Tag Name]? It will no longer appear in tag pickers, but existing patient records will retain it and it will remain available in reports." Confirm: "Archive Tag"
- If tag has 0 patients: archive immediately, no confirmation needed
- Hard delete only available on archived items with 0 patient records

### Empty State
- Icon (tag or label), heading "No tags yet", body "Create your first tag to start organizing patient records.", inline "+ Add Tag" button

### Seed Data
Pre-seed these active tags on first load / migration (assign distinct palette colors automatically):
VIP, PITA, Discharged, Surgical Candidate, Financing Needed, High Value, Recall Priority

## 3. TECHNICAL NOTES

### Database
**Tag model:** `id`, `name`, `color` (hex or palette key), `status` (ACTIVE | ARCHIVED), `practiceId`, `createdAt`, `updatedAt`
**PatientTag join table:** `patientId`, `tagId`, `assignedAt`
In Use count = PatientTag join count per tag.

### API Routes
- `GET /api/settings/tags` тАФ fetch all tags for practice, filter by status param
- `POST /api/settings/tags` тАФ create tag
- `PATCH /api/settings/tags/:id` тАФ update name/color
- `PATCH /api/settings/tags/:id/archive` тАФ archive tag
- `PATCH /api/settings/tags/:id/restore` тАФ restore archived tag
- `DELETE /api/settings/tags/:id` тАФ permanent delete (only if archived AND 0 patient records)

### Clerk Org Scoping
All tag operations must be scoped to the current Clerk Organization (practiceId derived from org context), consistent with the Clerk Organizations migration already implemented.

### Navigation
If a Settings nav item does not already exist in the main app nav, add one now so Settings is accessible from anywhere in the app.
