## Fix 1: Dynamic KPIs on Quotes Page

### Problem
KPIs on the Quotes page are static placeholders — they do not reflect real data and do not update when filters are applied.

### Fix
Wire all four KPI cards to live data that recalculates whenever filters change. The KPI query must respect all active filters (status, coordinator, date range, patient search).

### KPI Definitions

**Total Active Quotes**
- Count of quotes with status Draft or Sent
- Scoped to current filter state
- If coordinator filter is active: only quotes where coordinatorId matches

**Total Value Quoted**
- Sum of totals for Draft + Sent quotes
- Scoped to current filter state

**Acceptance Rate**
- Calculated at the **deal level**, not the quote level
- Formula:
  - Denominator: count of unique deals that have at least one quote matching current filters
  - Numerator: count of those deals where at least one quote has status = Converted
  - Rate = Numerator / Denominator × 100
- If coordinator filter is active: scope to deals where at least one quote is owned by that coordinator
- This ensures that if a deal had 3 quotes and 1 was converted, it counts as a converted deal (100% for that deal) — multiple unaccepted quotes on the same deal do not penalize the conversion rate
- Display as percentage with one decimal: "73.3%"
- If denominator is 0: show "—" instead of 0%

**Expiring Soon**
- Count of quotes with status Draft or Sent where expiration date is within the next 7 days
- Scoped to current filter state

### Implementation
- KPI data fetched from `GET /api/quotes/kpis` endpoint
- Pass current filter state as query parameters to this endpoint
- Re-fetch KPIs whenever any filter changes (debounced 300ms)
- Show subtle loading state on KPI cards during re-fetch (skeleton or opacity pulse)

## Fix 2: Dynamic KPIs on Invoices Page

### Problem
Same issue — Invoices page KPIs are static and do not update when filters change.

### Fix
Wire all four KPI cards to live data that recalculates whenever filters change.

### KPI Definitions

**Total Invoiced**
- Sum of totals for all non-void invoices
- Scoped to current filter state

**Amount Collected**
- Sum of all payment amounts recorded against invoices matching current filters
- Scoped to current filter state

**Outstanding Balance**
- Total Invoiced minus Amount Collected
- Scoped to current filter state

**Overdue**
- Count of invoices with status Sent or Partially Paid that are past their due date
- If no due date field exists on invoices yet, show count of Sent invoices older than 30 days as a proxy
- Scoped to current filter state

### Implementation
- KPI data fetched from `GET /api/invoices/kpis` endpoint
- Pass current filter state as query parameters
- Re-fetch whenever any filter changes (debounced 300ms)
- Subtle loading state during re-fetch

## Fix 3: Invoice View Page

### Problem
Clicking an invoice number or the eye icon on the Invoices page routes to the quote builder instead of an invoice view. No dedicated invoice view page exists.

### New Page: Invoice View
Route: `/invoices/[invoiceId]`

This is a read/edit view for an existing invoice. Layout mirrors the quote builder but is clearly labeled as an invoice.

### Header
- "Invoice" label with invoice number (e.g., **INV-2026-0001**) prominently displayed
- Status badge (Draft, Sent, Partially Paid, Paid, Void)
- Reference line: "Generated from [Q-2026-0006]" as a subtle clickable link back to the originating quote
- Action buttons (top right):
  - **Preview** — opens patient-facing invoice output in modal
  - **PDF** — downloads invoice PDF
  - **Mark as Sent** — changes status to Sent (if Draft)
  - **Void Invoice** — with confirmation dialog: "Voiding this invoice cannot be undone. Any recorded payments will be preserved for audit purposes. Continue?" Confirm / Cancel

### Invoice Details Section
- Patient (read-only, links to patient profile)
- Deal (read-only, links to deal on pipeline)
- Coordinator (editable dropdown)
- Created date (read-only)

### Line Items Section
- Fully editable — same interface as quote builder line items
- Changes save immediately or on explicit save action
- Recalculates totals on any change

### Totals Section
- Same as quote builder: subtotal, discount, tax, total
- Deposit section showing deposit amount
- Balance remaining shown prominently in green (if > 0, show in amber/orange)

### Payment History Section
Below totals, a dedicated **"Payments"** section:

**Header:** "Payment History" with **"+ Record Payment"** button on the right

**Payment table:**
| Date | Method | Amount | Reference | Recorded By | Actions |
|------|--------|--------|-----------|-------------|---------|
| Feb 23, 2026 | CareCredit | $3,700 | TXN-4892 | Amanda Torres | 🗑 |

**Running balance below table:**
```
Invoice Total:        $18,500.00
Total Paid:            $3,700.00
Balance Remaining:    $14,800.00
```

Balance Remaining shown in green if $0, amber if > $0.

Delete payment (trash icon): admin only, requires confirmation "Delete this payment record? This cannot be undone." Recalculates balance and status on delete.

### Record Payment Modal
Triggered by "+ Record Payment" button:
- Amount (currency input, pre-fills with balance remaining as hint)
- Payment Date (date picker, defaults to today)
- Payment Method (dropdown: Cash, Check, Credit Card, CareCredit, Cherry, PatientFi, Wire Transfer, Other)
- Reference Number (optional, label adapts: "Check #" for Check, "Transaction ID" for card/financing)
- Notes (optional textarea)
- Save Payment | Cancel

On save: payment recorded, totals update immediately, invoice status auto-updates.
