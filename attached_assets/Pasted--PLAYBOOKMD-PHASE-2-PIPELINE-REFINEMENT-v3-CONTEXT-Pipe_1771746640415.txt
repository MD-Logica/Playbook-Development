## FIX 2: ROTTING BADGE — MERGE WITH STATUS BADGE

The floating rotting badge (red circle top-right of card) is invisible against the white card background. Remove it entirely and replace with a merged approach.

### New approach: rotting modifies the status badge

When an opportunity is rotting, the existing status badge on the card changes to show rotting information:

**If the card already has a status badge (Past Due, New, Confirmed, etc.):**
- Keep the existing badge text
- Add "• Xd" after it in the same pill
- Example: "Past Due • 6d" or "New • 14d"
- The pill background becomes `#FEE2E2` (red tint)
- The pill text becomes `#991B1B` (dark red)
- The pill gets a `1px solid #FECACA` border

**If the card has no status badge:**
- Show a new badge: "Xd overdue"
- Same red tint styling as above

**Rotting threshold indicator on the badge:**
- 1–6 days overdue: amber styling (`#FEF3C7` bg, `#92400E` text, `#FDE68A` border) — "warming up"
- 7–13 days overdue: red styling (`#FEE2E2` bg, `#991B1B` text, `#FECACA` border) — "rotting"
- 14+ days overdue: deep red styling (`#FEE2E2` bg, `#7F1D1D` text, bold weight, `#FECACA` border) — "critical"

Remove all code related to the old floating circular badge. Remove `position: absolute` badge entirely.

### Card red tint when rotting:
Keep the subtle card background tint for 7+ day rotting cards:
- Card background: `rgba(239, 68, 68, 0.04)` instead of pure white
- Card border: `1px solid rgba(239, 68, 68, 0.2)`

## FIX 4: COLUMN SCROLLBAR — REPLACE WITH FADE GRADIENT

Remove the visible browser scrollbar from column lanes. Replace with a subtle fade-out gradient at the bottom of columns that have more cards than visible.

### Implementation:

The column lane container:
```css
.column-lane {
  overflow-y: auto;
  scrollbar-width: none;        /* Firefox */
  -ms-overflow-style: none;     /* IE/Edge */
}

.column-lane::-webkit-scrollbar {
  display: none;                /* Chrome/Safari */
}
```

Add a fade gradient overlay at the bottom of each column when scrollable:
- A pseudo-element or absolutely positioned div
- Background: `linear-gradient(to bottom, transparent, var(--column-bg))`
- Height: 40px
- Position: absolute, bottom of column, pointer-events none
- Only visible when the column has overflow (use JS to detect `scrollHeight > clientHeight`)

This creates a modern "content continues below" affordance without a visible scrollbar.
