## Context
Users currently have no way to manage their own account details from within PlaybookMD. This page gives every logged-in user access to update their personal information, manage credentials and two-factor authentication, and monitor their active sessions — all powered by Clerk under the hood.

---

## Settings Sidebar Update

Add **"Account"** as a new top-level item in the Settings sidebar nav, positioned above all other items since it's personal rather than practice-wide. It should have a subtle visual distinction from practice-level settings (e.g., a person/user icon).

---

## Page Layout

The My Account page is divided into three sections, displayed as distinct cards or sections with clear headings:

1. **Personal Information**
2. **Security & Authentication**
3. **Active Sessions**

---

## Section 1: Personal Information

### Fields
- **First Name** — text input, editable, syncs to Clerk user profile on save
- **Last Name** — text input, editable, syncs to Clerk user profile on save
- **Email Address** — text input, editable. Clerk requires email verification before the change takes effect — surface this clearly to the user ("A verification link will be sent to your new email address")
- **Phone Number** — text input with country code selector (same component as PP-C2), editable. Used for SMS-based 2FA and login.

### Save Behavior
A "Save Changes" button at the bottom of this section. Changes are not auto-saved. Show a success toast on save: "Profile updated successfully."

---

## Section 2: Security & Authentication

### Password
- "Change Password" button that triggers Clerk's password change flow
- If the user logs in via phone/SMS only and has no password set, show "Add Password" instead

### Two-Factor Authentication
- Show current 2FA status: Enabled / Not enabled
- If not enabled: show a subtle amber banner — "Your account is not protected by two-factor authentication. We strongly recommend enabling it." with an "Enable 2FA" button
- If enabled: show which methods are active with a green indicator
- Surface all 2FA methods Clerk supports:
  - SMS / Text message
  - Authenticator app (TOTP)
  - Passkey
  - Any other methods Clerk supports natively
- Each method can be added or removed independently
- Use Clerk's built-in UI components for the actual 2FA setup flows — do not build custom TOTP or passkey logic

### Note on Mandatory 2FA
Do not enforce mandatory 2FA in this prompt — that will be an admin-level setting added later. For now, only encourage it.

---

## Section 3: Active Sessions

### Current Sessions
Display a list of the user's active sessions pulled from Clerk. Each session shows:
- Device/browser info (e.g., "Chrome on macOS")
- Location if available (city/country)
- Last active timestamp
- **"This device"** label on the current session
- Individual "Sign out" button on each session except the current one

### Recent Sessions
Below active sessions, show the last 5 recently ended sessions (if Clerk provides this data) with:
- Device/browser info
- Last active timestamp
- Ended/expired status

This gives users visibility into unusual access patterns.

### Sign Out All Other Sessions
A **"Sign out all other devices"** button at the bottom of this section. Clicking shows a confirmation: "This will sign you out of all other active sessions. You will remain signed in on this device." Confirm / Cancel.

---

## Technical Notes

### Clerk Integration
- Use `useUser()` hook for reading and updating user profile data (name, email, phone)
- Use Clerk's `<UserProfile />` component or individual Clerk APIs for credential and 2FA management — wrap in PlaybookMD styling
- Use Clerk's Sessions API for listing and revoking sessions
- All changes must go through Clerk — do not store credentials or auth data in the PlaybookMD database
