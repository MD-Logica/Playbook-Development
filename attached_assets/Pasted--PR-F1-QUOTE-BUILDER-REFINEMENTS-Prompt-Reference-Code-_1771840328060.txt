## Fix 3: Patient-First Search Flow

### Current behavior (broken)
The form has a "Search deals..." input which is the wrong starting point.

### New behavior
Replace the Deal + Patient fields with:

**Step 1 — Patient search:**
- A single search input labeled "Patient" with placeholder "Search patients..."
- Searches by patient name, phone, or email
- Returns a dropdown list of matching patients with name and a subtle secondary detail (email or phone)
- Selecting a patient locks the patient field (shows their name, read-only)

**Step 2 — Deal selection:**
- Once a patient is selected, a "Deal" dropdown appears below
- Populated with that patient's open deals (non-closed)
- Each option shows deal title + current stage
- If the patient has only one open deal, auto-select it
- If no open deals exist, show: "No open deals — create a deal first" with a link to add one
- Selecting a deal locks the deal field (read-only)

When the quote builder is opened from within a deal's Finances tab, both patient and deal are pre-filled and locked — no search needed.

## Fix 8: Editorial PDF Design

### Design Direction
The patient-facing PDF should feel premium and editorial — think high-end medical publication or luxury brand proposal. Clean, generous white space, sophisticated typography, restrained use of color. Nothing that looks like generic software output.

### Typography
- **Headers** (practice name, section headings): Playfair Display or Cormorant Garamond (serif) — elegant, editorial weight
- **Body text** (line items, descriptions, contact info): Inter or DM Sans (sans-serif) — clean, highly legible
- Use font weight variation (light, regular, semibold) to create hierarchy without relying on color
- Load fonts via Google Fonts in the PDF generator

### Layout & Spacing
- Generous margins (at least 48px on all sides)
- Clear section separation with thin rules or white space — not heavy borders or boxes
- Line items table: minimal grid, light dividers between rows only (no heavy table borders)
- Right-aligned totals section with clear visual separation from line items

### Color Usage
- Neutral base: white background, near-black text (#1a1a1a or similar)
- Practice primary color used sparingly: quote number, section accent lines, deposit amount highlight, coordinator pill border
- Practice secondary color used even more sparingly: small accents only
- If no practice colors are configured, fall back to a sophisticated dark green (#1a3a2a) as default

### PDF Structure (top to bottom)

**Header section:**
- Practice logo (left-aligned, max height ~60px)
- Practice name in serif, large
- Right side: "Quote" label in large serif, Quote number below it in smaller sans-serif, muted
- Thin full-width rule below header

**Patient & Quote Info:**
- Two-column row: left = patient name + contact info | right = date issued, expiration date, coordinator name
- Subtle section, not visually heavy

**Line Items:**
- Section heading "Services" in serif
- Clean table: Item | Description | Qty | Price | Total
- Light row dividers only
- Bundle components hidden unless toggle is on
- Per-line-item discounts shown as a subtle line below the item in muted text
- Provider attribution NOT shown (internal only)

**Totals:**
- Right-aligned, clean
- Subtotal, discount (if any), tax, bold Total
- Thin rule above Total line

**Deposit Section:**
- Slightly set apart visually
- Label: "DEPOSIT TO SECURE YOUR DATE" in small caps or light weight caps
- Deposit amount in large, prominent type using practice primary color
- Percentage context below: "(20% of total)"

**Financing Section:**
- Subtle section: "Financing options available"
- Text mentions: CareCredit • Cherry • PatientFi
- Small, tasteful — not a banner

**Patient Notes:**
- If patient notes exist, shown in a lightly styled box or indented section
- Labeled "Notes"

**Coordinator Contact Pill:**
- At the very bottom of the page
- Pill-shaped container (rounded rectangle, subtle border in practice primary color or light gray)
- Contents: coordinator name (semibold) | email | phone
- Small and compact — like a digital business card
- Centered or left-aligned

**Footer:**
- Practice address/contact info in very small text
- "This quote is valid until [expiration date]"

### PDF Generation
Use a library like `@react-pdf/renderer` or `puppeteer` to generate the PDF server-side. Ensure fonts load correctly in the PDF context. PDF should be downloadable from a "Download PDF" button in the quote builder header actions area.
