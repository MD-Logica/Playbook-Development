## PRIORITY 1: FIX PATIENT PROFILE PAGE CRASH

The patient profile page at `/patients/[id]` is crashing within 1-2 seconds of loading with a page unload warning. Fix this first before anything else.

Likely causes to investigate:
1. An unhandled Promise rejection in a data fetching function
2. A Prisma query failing because a relation doesn't exist yet (Invoice, OpportunityForm, etc.)
3. A missing null check causing a runtime error on undefined data
4. An infinite re-render loop in a useEffect

Fix approach:
- Add try/catch error boundaries around ALL data fetching on the profile page
- Every API call must handle errors gracefully — return empty arrays/objects rather than throwing
- Add null safety to every data access: `patient?.name`, `opportunities?.length ?? 0`, etc.
- If Invoice or OpportunityForm tables don't exist yet, the queries must return empty arrays rather than crashing
- Add a top-level error boundary component so if the page does error, it shows a clean "Something went wrong — reload" message rather than a blank/crashing page

Test: navigate to `/patients/[any-id]` — page must load and stay stable without crashing.
