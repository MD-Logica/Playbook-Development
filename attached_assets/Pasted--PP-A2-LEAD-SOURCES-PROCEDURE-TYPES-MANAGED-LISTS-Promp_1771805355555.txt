## 1. LEAD SOURCES MANAGED LIST

### Purpose
Predefined, structured values assigned to patient/opportunity records to track lead origin. Replaces freeform text entry for reporting accuracy. Each source is marked Paid, Organic, or Relationship to enable marketing attribution reporting.

### Page Header
- Title: "Lead Sources"
- Subtitle: "Define the channels your practice uses to acquire leads. Used for marketing attribution and ROI reporting."
- Primary CTA: "+ Add Lead Source" (top right)

### Lead Source List Display
Each row shows:
- **Source name**
- **Channel type badge** ŌĆö pill badge: Paid (amber), Organic (green), Relationship (blue)
- **In Use count** ŌĆö number of records using this source
- **Status badge** ŌĆö only visible when Show Archived is toggled on
- **Actions** ŌĆö Edit, Archive

Same "Show Archived" toggle pattern as Tags.

### Add / Edit Lead Source ŌĆö Modal
Fields:
- **Source Name** ŌĆö text input, required, max 48 characters
- **Channel Type** ŌĆö segmented control with three options: Paid, Organic, Relationship
- **Save** / **Cancel**

Validation: Source name must be unique (case-insensitive).

### Alphabetical Sort
- "Sort AŌĆōZ" button above the table (left-aligned, secondary/ghost button style)
- One-click sorts the active list alphabetically
- Does not auto-sort on its own ŌĆö user controls when to apply

### Archive Behavior
Same as Tags: confirmation if in use, immediate archive if 0 records. Archived items hidden from all lead source pickers app-wide but preserved on historical records. Hard delete only on archived items with 0 records.

## 3. TECHNICAL NOTES

### Database Models

**LeadSource:** `id`, `name`, `channelType` (PAID | ORGANIC | RELATIONSHIP), `status` (ACTIVE | ARCHIVED), `sortOrder` (integer), `practiceId`, `createdAt`, `updatedAt`

**ProcedureType:** `id`, `name`, `category` (SURGICAL | NON_SURGICAL | SKINCARE | BODY | HAIR | OTHER), `status` (ACTIVE | ARCHIVED), `sortOrder` (integer), `practiceId`, `createdAt`, `updatedAt`

### API Routes

**Lead Sources:** `GET`, `POST` at `/api/settings/lead-sources` | `PATCH /:id`, `PATCH /:id/archive`, `PATCH /:id/restore`, `DELETE /:id`

**Procedure Types:** `GET`, `POST` at `/api/settings/procedure-types` | `PATCH /:id`, `PATCH /:id/archive`, `PATCH /:id/restore`, `DELETE /:id`

Delete only permitted when status is ARCHIVED and In Use count is 0.

### Wiring Into Existing App
Once both lists are built, update these to use structured values:
- Lead/opportunity creation form: replace freeform "Lead Source" text input with a dropdown sourced from active LeadSources
- Patient/opportunity record: replace freeform "Procedure of Interest" text input with a dropdown sourced from active ProcedureTypes

### Clerk Org Scoping
All operations scoped to current Clerk Organization (practiceId from org context), consistent with existing implementation.
