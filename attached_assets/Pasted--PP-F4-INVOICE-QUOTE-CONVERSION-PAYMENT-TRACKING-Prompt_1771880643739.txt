## Context
Quotes are proposals sent to patients. Once a patient accepts, the quote is converted to an invoice — the binding financial document. A deal can have multiple quotes but only one converted invoice at a time. Patients often pay in installments, so each invoice tracks a full payment history to enable reconciliation.

---

## Change 1: Quote → Invoice Conversion

### Entry Point
On any quote with status Draft, Sent, or Accepted, show an **"Accept & Convert to Invoice"** action button. This can appear:
- In the quote builder header actions
- On each quote card in the deal's Finances tab
- On the quote row in the patient profile Financial tab

### Conversion Flow
Clicking "Accept & Convert to Invoice" opens a confirmation modal:

**Heading:** "Convert Quote to Invoice?"

**Body summary:**
- Quote number
- Patient name
- Total amount
- Number of line items
- A note: "This will generate Invoice #INV-[next number] and mark this quote as Converted. Other open quotes for this deal will be marked as not accepted."

**Actions:** "Convert to Invoice" (primary green) | "Cancel"

### On Confirm
1. Quote status → Converted
2. Invoice created with all line items copied from the quote
3. All other quotes on the same deal that are Draft or Sent → status remains unchanged but visually marked as non-active (see Change 2)
4. Activity log entry on the deal: `{ action: "quote_converted", quoteId, quoteNumber, invoiceId, invoiceNumber }`

### Guard: One Active Invoice Per Deal
If a deal already has a non-void invoice, the "Accept & Convert to Invoice" button is hidden on all quotes for that deal. To convert a different quote, the existing invoice must first be voided.

## Change 2: Multi-Quote Visual Treatment on Deal

On the deal's Finances tab, where all quotes are listed:

### Converted Quote
- Green "Converted" badge
- Prominent, full opacity
- Shows linked invoice number as a clickable link: "→ INV-2026-0001"
- Slightly elevated visual treatment (subtle green left border or background tint)

### Draft / Sent Quotes (non-converted)
- Standard treatment, full opacity
- Show their status badge normally
- "Accept & Convert to Invoice" button available

### Declined Quotes
- Muted/grayed out (60% opacity)
- "Declined" badge in red
- Still expandable and readable
- No conversion action available

### Expired Quotes
- Muted/grayed out
- "Expired" badge in muted red
- Still readable
- No conversion action available

### General Rule
Nothing is ever hidden or deleted. All quotes remain accessible. Visual hierarchy makes it immediately clear which quote became the invoice.

## Change 3: Invoice Model & Creation

### Invoice Number
Auto-generated format: **INV-YYYY-####** (e.g., INV-2026-0001), sequential per practice.

### Invoice Fields
- Invoice number (auto-generated)
- Linked quote (read-only reference — "Generated from Q-2026-0001")
- Linked deal and patient (from quote)
- Coordinator (from quote, editable)
- Line items (copied from quote, fully editable — procedures can change up until surgery)
- Same totals logic as quote: subtotal, tax, discount, total
- Deposit amount (carried from quote)
- Status: Draft | Sent | Partially Paid | Paid | Void
- Internal notes
- Patient notes (shown on invoice PDF)
- Created date, updated date

### Status Auto-Management
- Default on creation: Draft
- When marked Sent: status → Sent
- When any payment recorded: if balance > 0 → Partially Paid, if balance = 0 → Paid
- Manual void option available (requires confirmation)

### Invoice Editing
Line items on an invoice are editable after creation since procedure plans can change close to surgery. Editing line items recalculates totals and updates balance remaining.

## Change 4: Payment Recording

### Payment Model
```prisma
model Payment {
  id              String        @id @default(cuid())
  invoiceId       String
  invoice         Invoice       @relation(fields: [invoiceId], references: [id])
  amount          Decimal
  paymentDate     DateTime
  method          PaymentMethod
  referenceNumber String?       // check number, transaction ID, etc.
  notes           String?
  recordedBy      String        // userId of staff who recorded it
  practiceId      String
  createdAt       DateTime      @default(now())
}

enum PaymentMethod {
  CASH
  CHECK
  CREDIT_CARD
  CARECREDIT
  CHERRY
  PATIENTFI
  WIRE_TRANSFER
  OTHER
}
```

### Recording a Payment
A **"+ Record Payment"** button appears on:
- Each invoice in the deal Finances tab
- Each invoice in the patient profile Financial tab
- The invoice view/editor itself

Clicking opens a modal:
- **Amount** — currency input, required. Pre-fills with balance remaining as a helper hint.
- **Payment Date** — date picker, defaults to today
- **Payment Method** — dropdown of PaymentMethod options
- **Reference Number** — text input, optional (label changes contextually: "Check #" for Check, "Transaction ID" for card/financing, "Reference #" for others)
- **Notes** — textarea, optional
- **Save Payment** | **Cancel**

On save: payment recorded, invoice totals update immediately, status auto-updates to Partially Paid or Paid.

### Payment History Display
On each invoice (in deal Finances tab, patient profile, and invoice view):

A **"Payments"** section below the invoice totals showing:

| Date | Method | Amount | Reference | Recorded By |
|------|--------|--------|-----------|-------------|
| Feb 23, 2026 | CareCredit | $3,700 | TXN-4892 | Amanda Torres |
| Mar 1, 2026 | Credit Card | $14,800 | — | Sofia Reyes |

Running balance shown clearly:
```
Invoice Total:      $18,500
Total Paid:          $3,700
Balance Remaining:  $14,800
```

Payments can be deleted (with confirmation) by admin users only. Deleting a payment recalculates balance and status.

## Technical Notes

### Database Models

```prisma
model Invoice {
  id              String        @id @default(cuid())
  invoiceNumber   String        // INV-YYYY-####
  quoteId         String?       @unique
  quote           Quote?        @relation(fields: [quoteId], references: [id])
  opportunityId   String
  opportunity     Opportunity   @relation(fields: [opportunityId], references: [id])
  coordinatorId   String
  status          InvoiceStatus @default(DRAFT)
  lineItems       InvoiceLineItem[]
  payments        Payment[]
  depositType     DepositType   @default(PERCENTAGE)
  depositValue    Decimal       @default(20)
  discountType    DiscountType?
  discountValue   Decimal?
  internalNotes   String?
  patientNotes    String?
  practiceId      String
  createdAt       DateTime      @default(now())
  updatedAt       DateTime      @updatedAt
}

model InvoiceLineItem {
  id               String   @id @default(cuid())
  invoiceId        String
  invoice          Invoice  @relation(fields: [invoiceId], references: [id])
  productServiceId String?
  name             String
  description      String?
  quantity         Decimal  @default(1)
  unitPrice        Decimal?
  hours            Decimal?
  providerId       String?
  discountType     DiscountType?
  discountValue    Decimal?
  sortOrder        Int      @default(0)
}

enum InvoiceStatus {
  DRAFT
  SENT
  PARTIALLY_PAID
  PAID
  VOID
}
```

### API Routes
- `POST /api/quotes/:id/convert` — convert quote to invoice
- `GET /api/invoices` — list all invoices (with filters)
- `GET /api/invoices/kpis` — KPI data for nav page
- `GET /api/opportunities/:id/invoices` — invoices for a deal
- `POST /api/invoices` — create invoice manually
- `PATCH /api/invoices/:id` — update invoice
- `PATCH /api/invoices/:id/status` — update status
- `POST /api/invoices/:id/payments` — record payment
- `DELETE /api/invoices/:id/payments/:paymentId` — delete payment (admin only)
- `GET /api/invoices/:id/pdf` — generate invoice PDF
