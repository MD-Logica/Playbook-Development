## Change 2: Saved Filters

### Data Model

```prisma
model SavedFilter {
  id          String   @id @default(cuid())
  name        String
  filters     Json     // serialized filter state
  visibility  FilterVisibility
  isStarred   Boolean  @default(false)
  createdBy   String   // userId
  practiceId  String
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}

enum FilterVisibility {
  PUBLIC   // visible to all practice members
  PRIVATE  // visible only to the creator
}
```

### Saving a Filter

When at least one filter is active, a "Save filter..." text link appears at the bottom of the filter panel, between Clear All and Apply Filters.

Clicking it expands an inline form directly within the panel (no modal):
- Text input: "Filter name" (required, max 48 characters)
- Toggle: "Private" / "Public" — default to Private
- Two buttons inline: "Save" (primary) and "Cancel" (text link)

On save: filter is stored, panel returns to normal state, saved filter appears in quick access row if starred.

### Saved Filters Section in Panel

Below the custom filter fields and above the Clear All / Apply Filters row, add a "Saved Filters" section with a subtle section label.

Each saved filter shows:
- Filter name
- Visibility badge ("Public" or "Private") — subtle, small
- Star icon to toggle starred/favorited status
- Edit icon (pencil) — opens the same inline form pre-populated with current name and visibility
- Delete icon (trash) — confirmation: "Delete [name]? This cannot be undone." Confirm / Cancel
- Clicking the filter name applies that saved filter set immediately

Only show saved filters the current user can see: their own private filters + all public filters.

### Starred Filters in Quick Access Row

Starred saved filters appear as pills in the quick filter row at the top of the panel, alongside My Leads, Rotting Only, High Value, Unassigned.

- Same pill style as existing quick filters
- Clicking applies the saved filter
- A small star icon on the pill differentiates them from system quick filters
- If no filters are starred, the quick filter row looks exactly as it does today

### API Routes
- `GET /api/pipeline/saved-filters` — fetch saved filters visible to current user
- `POST /api/pipeline/saved-filters` — create saved filter
- `PATCH /api/pipeline/saved-filters/:id` — update name, visibility, isStarred
- `DELETE /api/pipeline/saved-filters/:id` — delete (only if createdBy = current user)
