## FIX 2: NUMBER COUNT-UP ANIMATION

When KPI data loads, animate each number counting up from 0 to its final value. This is purely visual — it makes the dashboard feel alive when you land on it.

Implement this hook in `hooks/useCountUp.ts`:

```typescript
import { useState, useEffect } from 'react';

export function useCountUp(target: number, duration: number = 800, delay: number = 0) {
  const [count, setCount] = useState(0);
  
  useEffect(() => {
    if (target === 0) { setCount(0); return; }
    
    const timeout = setTimeout(() => {
      const start = Date.now();
      const timer = setInterval(() => {
        const elapsed = Date.now() - start;
        const progress = Math.min(elapsed / duration, 1);
        const eased = 1 - Math.pow(1 - progress, 3); // ease-out cubic
        setCount(Math.round(target * eased));
        if (progress >= 1) {
          setCount(target);
          clearInterval(timer);
        }
      }, 16);
      return () => clearInterval(timer);
    }, delay);
    
    return () => clearTimeout(timeout);
  }, [target, duration, delay]);
  
  return count;
}
```

Apply to each KPI card with staggered delays:
- Card 1: `useCountUp(todayTotal, 600, 0)`
- Card 2: `useCountUp(todayConfirmed, 600, 80)`
- Card 3: `useCountUp(todayPending, 600, 160)`
- Card 4: `useCountUp(rottingLeads, 600, 240)`

For cards showing "—" at zero, still run the hook but conditionally render "—" if target === 0.

## FIX 3: PAGE LOAD STAGGER ANIMATION


Apply inline styles or className assignments with these exact delays:
- Page header: `animation: fadeUp 280ms ease-out 0ms both`
- KPI card 1: `animation: fadeUp 280ms ease-out 60ms both`
- KPI card 2: `animation: fadeUp 280ms ease-out 110ms both`
- KPI card 3: `animation: fadeUp 280ms ease-out 160ms both`
- KPI card 4: `animation: fadeUp 280ms ease-out 210ms both`
- Left column (schedule): `animation: fadeUp 280ms ease-out 270ms both`
- Right column (alerts + quotes): `animation: fadeUp 280ms ease-out 310ms both`

The `both` fill-mode is critical — it keeps elements invisible before their animation starts and locked in place after. Without it elements will flash visible then animate.
