## Change 3: Income Categories Page

### Purpose
Income Categories are a user-configurable hierarchical tree used to classify products and services for financial reporting. Think of them as a chart of accounts — practices organize their revenue into categories they can drill down into for reporting purposes.

### Page Header
- Title: "Income Categories"
- Subtitle: "Organize your services into categories for financial reporting. Think of these as your chart of accounts — the more accurately you categorize, the more useful your reports will be."
- Two buttons top right: **"Load Playbook Template"** (secondary) and **"+ Add Category"** (primary)

### Tree Display
Render categories as an indented, expandable tree. Each node shows:
- **Expand/collapse chevron** if it has children
- **Category name**
- **Item count** — number of products/services assigned to this category (e.g., "4 items") — show only if > 0
- **Add child** icon — small + icon to add a subcategory directly under this node
- **Edit** icon — pencil to rename inline
- **Archive** icon — archive this category
- **Drag handle** — for reordering within the same level

Top-level categories have a slightly bolder treatment. Child categories are indented proportionally to their depth.

### Adding Categories
- **"+ Add Category"** button adds a new top-level category — inline input appears at the bottom of the top-level list
- **Add child icon** on any node adds a subcategory — inline input appears as a child of that node
- Press Enter to confirm, Escape to cancel
- Category names must be unique within the same parent level (case-insensitive)

### Renaming
Click edit icon → name becomes inline editable → Enter to confirm, Escape to cancel.

### Reordering
Drag handle allows reordering within the same level. Cannot drag a category to become a child of a different parent via drag — that would require a separate "move" action (defer this to later, keep it simple for now). Order persists to database on drop.

### Archive Behavior
Same pattern as other managed lists:
- If category has 0 items assigned AND 0 children: archive immediately
- If category has items assigned OR has children: show confirmation — "Archiving [Name] will also archive all subcategories and unassign [X] products/services from this category. They can be reassigned after. Continue?"
- Archived categories hidden from tree by default
- "Show Archived" toggle above tree (right-aligned) reveals archived nodes with muted styling and Restore option
- Hard delete only available on archived categories with 0 items and 0 children

### Empty State
If no categories exist:
- Icon + "No income categories yet"
- Body: "Set up your chart of accounts to enable financial reporting. You can build your own structure or load our recommended template for aesthetic practices."
- Two buttons: "Load Playbook Template" and "+ Add Category"

## Technical Notes

### Database Model

```prisma
model IncomeCategory {
  id          String    @id @default(cuid())
  name        String
  parentId    String?
  parent      IncomeCategory?  @relation("CategoryTree", fields: [parentId], references: [id])
  children    IncomeCategory[] @relation("CategoryTree")
  sortOrder   Int       @default(0)
  status      CategoryStatus @default(ACTIVE)
  practiceId  String
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
}

enum CategoryStatus {
  ACTIVE
  ARCHIVED
}
```

### Tax Settings on Practice Model
Add `defaultTaxRate` (Decimal, nullable) to the Practice model.

### API Routes
- `GET /api/settings/income-categories` — fetch full tree for practice
- `POST /api/settings/income-categories` — create category (with optional parentId)
- `PATCH /api/settings/income-categories/:id` — rename, reorder
- `PATCH /api/settings/income-categories/:id/archive` — archive
- `PATCH /api/settings/income-categories/:id/restore` — restore
- `DELETE /api/settings/income-categories/:id` — permanent delete (archived, 0 items, 0 children only)
- `POST /api/settings/income-categories/load-playbook` — load template
- `GET /api/settings/tax` — get tax settings
- `PATCH /api/settings/tax` — update default tax rate

### Clerk Org Scoping
All operations scoped to current Clerk Organization (practiceId from org context).
