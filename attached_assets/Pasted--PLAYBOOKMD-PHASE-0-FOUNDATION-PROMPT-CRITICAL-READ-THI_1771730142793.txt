## TECH STACK — NON-NEGOTIABLE. DO NOT DEVIATE.

- **Framework:** Next.js 15 with App Router
- **Database:** PostgreSQL via **Prisma ORM** exclusively. All database access goes through Prisma. No raw SQL. No platform-native database tools.
- **Database Provider:** Neon (serverless Postgres). Connection string via `DATABASE_URL` environment variable.
- **Authentication:** Clerk. Use Clerk's Next.js SDK only. No custom auth. No platform-native auth.
- **Styling:** Tailwind CSS
- **Form Validation:** Zod
- **State Management:** TanStack Query (React Query) + Zustand — install now, configure later
- **Package Manager:** npm

**Required packages to install:**
```
npm install @prisma/client prisma @clerk/nextjs @tanstack/react-query zustand zod
npx prisma init
```

## ARCHITECTURE RULES — ABSOLUTE. FOLLOW WITHOUT EXCEPTION.

1. All credentials and service connections via environment variables only. Never hardcoded.

2. Create a `.env.example` file listing every required variable with placeholder values and a comment explaining each one.

3. Build a `/lib` abstraction layer:
   - `/lib/db.ts` — exports the Prisma client singleton
   - `/lib/auth.ts` — exports Clerk helper utilities
   - No component or API route ever imports Prisma or Clerk directly. They always import from `/lib`.

4. All data mutations and queries go through Next.js API routes in `/app/api/`. No direct database calls from client components. Ever.

5. No platform-native databases, key-value stores, or Replit-specific packages. This codebase must run anywhere Node.js runs.

6. Create a `/prisma/seed.ts` file with comprehensive dummy data as specified below. Seeding must be runnable with `npx prisma db seed`.

## PRISMA SCHEMA

Design the schema to support the full application roadmap, not just Phase 0. Every model below must be included even if not all fields are used immediately. Use proper relations, indexes on foreign keys and commonly queried fields, and soft deletes (`deletedAt DateTime?`) on all major entities.

### Practice
The top-level tenant. Everything belongs to a practice.
```
id, name, address, city, state, phone, email, timezone, logoUrl,
settings (Json), createdAt, updatedAt, deletedAt
```

### User
Staff members. Linked to Clerk via clerkId.
```
id, clerkId (unique), practiceId, firstName, lastName, email, phone,
role (enum: OWNER, MANAGER, PROVIDER, COORDINATOR, FRONT_DESK, MARKETING),
avatarUrl, isActive, createdAt, updatedAt, deletedAt
```

### Patient
The core patient/contact record.
```
id, practiceId, firstName, lastName, email, phone, dateOfBirth, gender,
address, city, state, zip,
referralSource (enum: GOOGLE_ADS, FACEBOOK_ADS, INSTAGRAM, WORD_OF_MOUTH, WALK_IN, WEBSITE, PHONE, OTHER),
referredByPatientId (self-relation),
status (enum: LEAD, ACTIVE, INACTIVE, VIP, FLAGGED),
isMinor (Boolean), parentGuardianId (self-relation),
photoConsentStatus (enum: NONE, INTERNAL_ONLY, MARKETING_APPROVED),
isVip, vipAlias, notes, tags (String[]),
createdAt, updatedAt, deletedAt
```

### Pipeline
Customizable pipelines (Surgical, Non-Surgical, etc.)
```
id, practiceId, name, description, isDefault, isActive, order,
createdAt, updatedAt
```

### PipelineStage
Stages within a pipeline.
```
id, pipelineId, name, order, color,
rotDaysWarning (Int — days before card turns yellow),
rotDaysCritical (Int — days before card turns red),
isWon, isLost, createdAt, updatedAt
```

### Opportunity
A lead/deal moving through a pipeline. The central CRM object.
```
id, practiceId, patientId, pipelineId, stageId, assignedToId (User),
title, value (Decimal),
conversionType (enum: NET_NEW, UPSELL, CROSS_SELL),
referralSource, referredByPatientId,
lastActivityAt, stageEnteredAt,
isWon, isLost, lostReason, wonAt, lostAt,
createdAt, updatedAt, deletedAt
```

### Activity
The global timeline. Every touchpoint for a patient or opportunity logs here.
```
id, practiceId, patientId, opportunityId (optional), userId,
type (enum: NOTE, CALL, SMS, EMAIL, APPOINTMENT, QUOTE, INVOICE, PAYMENT,
           CONSENT, DOCUMENT, PHOTO, TASK, STAGE_CHANGE, SYSTEM),
body, metadata (Json), isInternal,
createdAt, updatedAt
```

### Appointment
```
id, practiceId, patientId, opportunityId (optional), providerId (User),
createdById (User),
appointmentType (enum: CONSULT, FOLLOW_UP, TREATMENT, SURGERY, POST_OP, OTHER),
title, startTime, endTime,
status (enum: PENDING, CONFIRMED, COMPLETED, CANCELLED, NO_SHOW, HOLD),
location, notes, confirmationSentAt, reminderSentAt,
cancelledAt, cancelReason, noShowFee (Decimal),
createdAt, updatedAt, deletedAt
```

### AppointmentHold
Temporary holds with pay-to-confirm links.
```
id, appointmentId, expiresAt,
paymentRequirement (enum: NONE, CARD_ON_FILE, DEPOSIT, FULL_PAYMENT),
secureToken (unique), completedAt, createdAt
```

### Quote
```
id, practiceId, patientId, opportunityId, createdById, assignedToId,
status (enum: DRAFT, SENT, VIEWED, ACCEPTED, DECLINED, EXPIRED),
subtotal (Decimal), discountAmount (Decimal), discountReason, total (Decimal),
expiresAt, sentAt, viewedAt, acceptedAt, declinedAt, notes,
createdAt, updatedAt, deletedAt
```

### QuoteLineItem
```
id, quoteId, procedureId (optional), description, quantity,
unitPrice (Decimal), discountAmount (Decimal), total (Decimal),
providerId (optional), coordinatorId (optional), order,
createdAt, updatedAt
```

### Invoice
```
id, practiceId, patientId, quoteId (optional),
```

### Payment

### TreatmentPlan
