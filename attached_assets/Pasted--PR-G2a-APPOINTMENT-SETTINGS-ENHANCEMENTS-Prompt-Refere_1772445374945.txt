## Change 1: Prisma Schema Additions

Add the following models and enums to `prisma/schema.prisma` **after the existing `AppointmentSubcategory` model** at the bottom of the file.

```prisma
model AppointmentCustomStatus {
  id           String    @id @default(cuid())
  practiceId   String
  label        String
  color        String    @default("#6B7280")
  isSystem     Boolean   @default(false)
  isTerminal   Boolean   @default(false)
  sortOrder    Int       @default(0)
  createdAt    DateTime  @default(now())
  updatedAt    DateTime  @updatedAt

  practice     Practice  @relation(fields: [practiceId], references: [id])

  @@unique([practiceId, label])
  @@index([practiceId])
}

model ReadinessIndicator {
  id              String                   @id @default(cuid())
  practiceId      String
  label           String
  icon            String                   @default("check-circle")
  integrationType ReadinessIntegrationType @default(MANUAL)
  integrationKey  String?
  isEnabled       Boolean                  @default(true)
  sortOrder       Int                      @default(0)
  createdAt       DateTime                 @default(now())
  updatedAt       DateTime                 @updatedAt

  practice        Practice                 @relation(fields: [practiceId], references: [id])

  @@unique([practiceId, label])
  @@index([practiceId])
}

enum ReadinessIntegrationType {
  MANUAL
  STRIPE
  DOCUSIGN
  PANDADOC
}

model InternalBlockType {
  id          String    @id @default(cuid())
  practiceId  String
  name        String
  color       String    @default("#6B7280")
  isSystem    Boolean   @default(false)
  sortOrder   Int       @default(0)
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt

  practice    Practice  @relation(fields: [practiceId], references: [id])

  @@unique([practiceId, name])
  @@index([practiceId])
}

model AppointmentAttendee {
  id            String      @id @default(cuid())
  appointmentId String
  userId        String
  isPrimary     Boolean     @default(false)
  createdAt     DateTime    @default(now())

  appointment   Appointment @relation(fields: [appointmentId], references: [id], onDelete: Cascade)
  user          User        @relation(fields: [userId], references: [id])

  @@unique([appointmentId, userId])
  @@index([appointmentId])
  @@index([userId])
}
```

## Change 3: API Routes

### `GET /api/settings/appointment-statuses`
Return all statuses for the practice sorted by `sortOrder`. If the count is 0, seed defaults first using `prisma.appointmentCustomStatus.createMany()`, then return them.

### `POST /api/settings/appointment-statuses`
Create a new custom status. Validate:
- `label` required, max 40 chars
- `label` must be unique within the practice
- `color` must be a valid hex color string (matches `/^#[0-9A-Fa-f]{6}$/`)
- Cannot set `isSystem: true` via POST — ignored if provided
Returns the created record.

### `PATCH /api/settings/appointment-statuses/[id]`
Accepts: `label`, `color`, `sortOrder`, `isTerminal`.
- If the status has `isSystem: true`: only `color` and `sortOrder` may be changed. Attempts to change `label` or `isTerminal` on system statuses return 400: `"System status labels cannot be modified."`
- Cannot change `isSystem` flag at all.

### `DELETE /api/settings/appointment-statuses/[id]`
If `isSystem: true`: return 400 `"System statuses cannot be deleted."` Otherwise, hard delete.

### `POST /api/settings/appointment-statuses/reorder`
Body: `{ ids: string[] }` — ordered array of status IDs. Update `sortOrder` for each in a `prisma.$transaction`, where index in the array becomes the new `sortOrder`.

---

### `GET /api/settings/readiness-indicators`
Return all indicators for the practice sorted by `sortOrder`. Seed defaults if none exist.

### `POST /api/settings/readiness-indicators`
Validate: `label` required, max 50 chars, unique within practice. `icon` must be one of: `check-circle`, `dollar-sign`, `file-text`, `clipboard`, `shield`, `heart`, `user-check`, `calendar-check`, `camera`, `pill`, `stethoscope`. `integrationType` must be a valid `ReadinessIntegrationType` enum value. Returns created record.

### `PATCH /api/settings/readiness-indicators/[id]`
Accepts: `label`, `icon`, `isEnabled`, `sortOrder`, `integrationType`. No system-lock — all indicators (including defaults) can be fully edited.

### `DELETE /api/settings/readiness-indicators/[id]`
Hard delete. No restrictions.

### `POST /api/settings/readiness-indicators/reorder`
Same pattern as status reorder — body `{ ids: string[] }`, updates `sortOrder` per index.

---

### `GET /api/settings/internal-block-types`
Return all block types for the practice sorted by `sortOrder`. Seed defaults if none exist.

### `POST /api/settings/internal-block-types`
Validate: `name` required, max 40 chars, unique within practice. Cannot set `isSystem: true` via POST.

### `PATCH /api/settings/internal-block-types/[id]`
Accepts: `name`, `color`, `sortOrder`. If `isSystem: true`: only `color` and `sortOrder` may be changed. Changing `name` on a system type returns 400: `"System block type names cannot be modified."`

### `DELETE /api/settings/internal-block-types/[id]`
If `isSystem: true`: return 400 `"System block types cannot be deleted."` Otherwise, hard delete.
