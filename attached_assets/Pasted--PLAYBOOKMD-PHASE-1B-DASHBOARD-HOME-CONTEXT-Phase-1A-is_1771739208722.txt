## CONTEXT

Phase 1A is complete. The app shell, dark sidebar, navigation, top bar, and brand design system are all in place. The color system, typography, and CSS variables are defined in `globals.css`.

This is Phase 1B. You are building the **real dashboard home page** — replacing the placeholder stat cards with a genuinely useful, time-aware morning view for a front desk manager or patient coordinator. ## API ROUTES REQUIRED

Create the following API routes to power the dashboard. All routes must scope queries to `practiceId` (use `prisma.practice.findFirst()` for now as the practice resolver):

**`GET /api/dashboard/kpis`**
Returns:
```json
{
  "todayTotal": 5,
  "todayConfirmed": 3,
  "todayPending": 2,
  "rottingLeads": 4
}
```

**`GET /api/dashboard/today-appointments`**
Returns today's appointments with patient name, provider name, type, status, startTime. Ordered by startTime.

**`GET /api/dashboard/rotting-leads`**
Returns opportunities where lastActivityAt < 7 days ago. Include patient name, stage name, assigned user first name, lastActivityAt. Limit 5.

**`GET /api/dashboard/pending-quotes`**
Returns quotes where status IN [SENT, VIEWED]. Include patient name, total, sentAt, viewedAt. Limit 4. Ordered by sentAt ascending.

Use TanStack Query (`useQuery`) on the client to fetch all four endpoints. Show skeleton loaders while loading — not spinners. Use a simple pulse animation on placeholder rectangles that match the shape of the content they're replacing.

## ANIMATIONS & MICRO-INTERACTIONS

**Page load sequence:**
1. Header fades in first (0ms delay, 250ms)
2. KPI cards stagger in (starting at 100ms, 60ms apart)
3. Both columns fade up together (300ms delay, 300ms)

**Row hover states:** All list rows get `background-color` transition on hover — smooth 150ms, no jumping

**Number counting animation on KPI cards:**
When the data loads, the numbers count up from 0 to their final value over 600ms with an ease-out curve. Keep it subtle — this should feel satisfying, not gimmicky.

**Status chip pulse:**
Any PENDING appointment status chip gets a very subtle pulse animation — the amber dot softly pulses (opacity 1 → 0.5 → 1) on a 2s loop. This draws attention to unconfirmed appointments without being aggressive.

**Days-since badge:**
For leads that are 14+ days old (danger/red), the "🔥 15d ago" text has a very subtle shake animation on hover — just 2–3px horizontal, 200ms — like a gentle nudge to act.
